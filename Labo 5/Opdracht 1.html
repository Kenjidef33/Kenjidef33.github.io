Bij de DOM structuur voor deze opdracht zijn de siblings van het < ul > element de < h1 > en de < p > elementen, doordat ze allemaal binnen in hetzelfde body zitten in deze DOM-structuur. < h1 > en < p > zijn dus broertjes van < ul >

Bij de tweede < li > element zijn de ancestors de elementen die zich bovenaan bevinden in het html document. Deze zijn dus < ul >, < body > en < html &gt. HTML bevindt zich ook altijd bovenaan het document en zal dus altijd een ancestor zijn van de andere elementen.

Descendants of nakomelingen zijn alle elementen die zich binnenin een specifieke element bevinden. Voor het element < body > zijn de descendants < h1 >, < p >, < ul > en alle 3 de < li > elementen ook.

Elk geldig HTML-document begint met een < html > element als het root-element. Dit betekent dat alle andere elementen binnen de < html > tag als children worden beschouwd. Dus in een geldig HTML-document heeft geen enkel element geen parent.

Niet elk element heeft child elementen, maar de meeste wel. De & lt h1 >, < p >, < ul >, en & lt li > elementen in je voorbeeld hebben child elementen, zoals tekst of andere HTML-elementen. Elementen zoals < img > kunnen bijvoorbeeld geen andere elementen bevatten en hebben geen child elementen. Het varieert afhankelijk van het type element en de inhoud ervan.
